---
description: Создать PR develop → main (релиз в production) — команда ТОЛЬКО для разработчика
---

Релиз в production запускает только разработчик. Сначала проверь, кто пользователь:

0. `gh api user -q .login` — если результат НЕ `webkoth` (или gh не авторизован),
   сообщи: «Релиз в production запускает только разработчик. Сообщи ему, что твои
   фичи проверены в песочнице и готовы к релизу» — и остановись.

Дальше (только разработчик):

1. `git fetch origin`
2. Собери список изменений с прошлого релиза:
   `git log origin/main..origin/develop --oneline --no-merges`
   Если список пуст — сообщи, что релизить нечего, и остановись.
3. Создай PR:
   `gh pr create --base main --head develop --title "release: <краткая суть пачки>" --body "<маркированный список изменений человеческим языком + список коммитов>"`
   В body обязательно отметь, есть ли в пачке миграции БД (посмотри
   `git diff origin/main..origin/develop --stat -- prisma/migrations/`) и деструктивные
   операции в них.
4. Сообщи ссылку на PR. После ревью и merge production обновится автоматически
   (workflow deploy-prod).

Сам PR не мержи без явного решения разработчика.

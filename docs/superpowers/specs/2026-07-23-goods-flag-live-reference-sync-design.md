# Галочка «оплата за товар» в справочнике ДДС + полный список справочников из живой 1С

Дата: 2026-07-23. Статус: одобрено Антониной (устно в сессии), реализация — по плану.

## Проблема

1. В разделе «Справочники» видно не все статьи: синк справочников из 1С работает
   в демо-режиме (`ONEC_ODATA_MODE=fixture`), реальная база (203 статьи ДДС,
   7 статей БДР, счета) не подтягивается.
2. Вкладка настроек «Статьи для отправки» (`/settings/cash-flow-items`) ведёт
   отдельный список статей ДДС, наполняемый из текста заявок на оплату, — он
   дублирует справочник ДДС. Единственная её ценность — галочка «оплата за
   товар», по которой синк создаёт черновики отправки платёжек.

## Решение (вариант А, выбран из трёх)

Признак «оплата за товар» становится локальным полем статьи справочника ДДС.
Отдельная таблица и вкладка удаляются. Синк переключается на живую 1С.

### 1. Данные и миграция

- `Article` получает поле `isGoods Boolean @default(false)` — с комментарием
  в схеме: «локальный признак finflow, не из 1С; синк не трогает».
- Синк справочников (`lib/sync/run-reference-sync.ts`) поле `isGoods`
  не читает и не пишет ни при создании, ни при обновлении, ни при архивации.
- Миграция (одна, деструктивная):
  1. добавить колонку `is_goods` в `articles`;
  2. перенести галочки: `UPDATE articles SET is_goods = true` для статей
     `kind = CASHFLOW`, чьё `name` совпадает с записью
     `cash_flow_item_settings.is_goods = true`;
  3. удалить таблицу `cash_flow_item_settings` и модель `CashFlowItemSetting`.
- Потеря данных допустима и озвучена: галочки, чьё название не нашлось
  в справочнике, пропадают. После перехода на живую 1С справочник полный,
  реальные статьи находятся по названию.

### 2. Интерфейс

- Страница `/reference/cashflow-items` («Статьи ДДС»): у конечных активных
  статей (не папок) — переключатель «оплата за товар». Переключение — server
  action по паттерну `useActionState`, право — существующее
  `manage_cash_flow_items` (проверка в action). Без права переключатель
  виден, но недоступен (disabled) — состояние галочек видно всем.
- Подпись/подсказка на странице: «По статьям с признаком „оплата за товар“
  синк создаёт черновики отправки платёжек».
- Удаляются: `app/settings/cash-flow-items/` (страница, таблица, actions),
  пункт «Статьи для отправки» в сайдбаре (`components/app-sidebar.tsx`).
- Право `manage_cash_flow_items` остаётся, меняется только место применения.

### 3. Логика отправки платёжек

`lib/sync/sync-dispatch.ts`:

- Шаг 1 (upsert названий статей из заявок в `cashFlowItemSetting`) удаляется.
- Список «товарных» названий берётся из справочника:
  `article.findMany({ where: { kind: "CASHFLOW", isGoods: true, isActive: true } })`.
- Сопоставление заявка → статья остаётся по названию (`request.cashFlowItem` —
  строка из DWH, идентификатора 1С в заявках нет; источник названий один — 1С).
- Статья заявки не найдена в справочнике или без галочки → черновик
  не создаётся (текущее поведение сохранено).

### 4. Живая 1С

- `lib/integrations/one-c-odata-http.ts`:
  - имя набора БДР: `Catalog_СтатьиДоходовИРасходов` →
    **`Catalog_RSФП_СтруктураБДР`** (реальное имя, проверено 2026-07-22);
  - для БДР не запрашивать поля `ВидДвижения` и `Комментарий` — справочник
    их не отдаёт; `flow` у статей БДР остаётся `null` (решение 2026-07-22:
    вид движения статьям БДР не нужен).
- Проверка живого синка — локально (`ONEC_ODATA_MODE=real`, доступ
  к `192.168.79.250` из офисной сети есть, логин `ClaudeOR` проверен):
  ожидаем 203 статьи ДДС (43 папки, включая архивную ветку «Не используются
  с 01.07.2023» — она приходит как обычные записи), 7 статей БДР, счета.
- Песочница (VPS 161.104.50.20): **риск** — 1С во внутренней сети офиса,
  с внешнего VPS может быть недоступна. Порядок: проверить доступность с VPS;
  если недоступна — эскалация Минасу (туннель/проброс), песочница до решения
  остаётся на `fixture`. Переключение env на VPS — вне кода, отдельным шагом.

### 5. Тесты

- Демо-данные `one-c-odata-fixture.ts` дополняются статьями с названиями,
  совпадающими с `dwh-fixture.ts` (минимум «Оплата поставщикам за товар»), —
  иначе e2e-сценарий отправки платёжек не сойдётся по названию.
- Юнит-тесты `lib/domain/reference/sync-diff` не меняются (поле локальное,
  в дифф не входит).
- E2e `tests/e2e/reference.spec.ts`: добавить сценарий переключателя
  «оплата за товар» на странице «Статьи ДДС».
- Тесты и код, завязанные на `CashFlowItemSetting` и `/settings/cash-flow-items`,
  удаляются/обновляются.

## Вне объёма

- Новые виды справочников (контрагенты, организации и т.п.) — не в этой задаче.
- Связь ДДС → БДР (реквизит в 1С добавляет Антонина) — отдельная задача.
- Запуск модуля отправки платёжек в production — не затрагивается.

## Отклонённые варианты

- **Б: отдельная таблица флагов** со ссылкой на статью — чище идейно, но
  лишний код ради одной галочки; вернёмся, если локальных признаков станет много.
- **В: список кодов статей в конфиге** — правки только через разработчика,
  не гибко.

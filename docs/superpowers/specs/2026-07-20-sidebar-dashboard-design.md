# Сайдбар и дашборд — дизайн

Дата: 2026-07-20. Статус: утверждён в brainstorming-сессии.

## Цель

Заменить временную навигацию (шапка + главная страница со списком ссылок) на
постоянный каркас приложения: сайдбар слева со всеми разделами и дашборд на `/`
с ключевыми показателями: остатки на счетах, заявки на оплату, отправка
платёжек, динамика движения денег.

## Компоненты и зависимости

- Из реестра shadcn ставятся стоковые компоненты: `sidebar`, `chart`,
  `toggle-group` (одной командой `npx shadcn@latest add sidebar chart toggle-group`).
  Sidebar притянет свои зависимости (sheet, tooltip, skeleton) в `components/ui/` —
  они стоковые, руками не редактируются.
- **Новая npm-зависимость: `recharts`** — приходит вместе с компонентом `chart`;
  это официальный рантайм shadcn charts (единственный разрешённый способ строить
  графики по правилам проекта). Другие новые зависимости не добавляются.
- Примитивы — Base UI (`@base-ui/react`): в сайдбаре ссылки рендерятся через
  `Button` с пропом `render` и `nativeButton={false}` (не `asChild`).

## Каркас (AppShell)

- Новый серверный компонент `components/app-shell.tsx` оборачивает `{children}`
  в корневом `app/layout.tsx`.
- Логика видимости — как у нынешней шапки: `getCurrentUser()` вернул `null`
  (страница `/login`) → children рендерятся без каркаса; иначе — сайдбар слева
  (`SidebarProvider` + `Sidebar`) и контент в `SidebarInset` с тонкой шапкой
  сверху (кнопка сворачивания `SidebarTrigger`).
- Файлы страниц не переезжают, структура `app/` не меняется (route-группы не вводим).
- `components/app-header.tsx` удаляется; его содержимое (имя, бейдж роли,
  «Сменить пароль», «Выйти») переезжает в футер сайдбара.
- Мобильная версия — стоковая у shadcn Sidebar: на узком экране меню в
  выдвижной панели (Sheet), кнопка-гамбургер в шапке.

## Сайдбар: состав и роли

Группы и пункты (сверху вниз), у каждого пункта иконка lucide:

| Группа      | Пункт                | Роут                       | Кто видит |
| ----------- | -------------------- | -------------------------- | --------- |
| Обзор       | Дашборд              | `/`                        | все |
| Операции    | Транзакции           | `/transactions`            | все |
| Операции    | Заявки на оплату     | `/requests`                | все |
| Операции    | Отправка платёжек    | `/dispatch`                | все |
| Справочники | Справочники          | `/reference`               | все |
| Настройки   | Пользователи         | `/settings/users`          | `can(role, "manage_users")` — собственник |
| Настройки   | Статьи для отправки  | `/settings/cash-flow-items`| `can(role, "manage_cash_flow_items")` — собственник, бухгалтер |
| Настройки   | Светофор             | `/settings/verdict`        | `can(role, "manage_verdict_settings")` — собственник |

- Видимость — только через существующий `can()` из `lib/domain/permissions.ts`;
  новых прав не вводим. Если в группе не осталось видимых пунктов (у читателя —
  «Настройки»), группа скрывается целиком.
- Скрытие пункта — только косметика: доступ на страницах по-прежнему
  контролируют их собственные проверки (`notFound()` при отсутствии права).
- Активный пункт подсвечивается. Список пунктов собирает серверный компонент
  (фильтрация по роли), подсветку делает клиентский подкомпонент с
  `usePathname`; данные попадают в него только через пропсы.
- Футер сайдбара: имя пользователя, бейдж роли (`ROLE_LABELS`), ссылка
  «Сменить пароль» (`/settings/password`), кнопка «Выйти» (форма с существующим
  server action `logout`).

## Дашборд `/`

Заменяет нынешний список ссылок. Server component; все запросы —
параллельно через `Promise.all`. Структура сверху вниз:

### 1. Ряд из четырёх stat-карточек (Card)

- **«На счетах»** — итог в ₽ по всем счетам + число счетов.
- **«На согласовании»** — количество заявок с `approvalStatus = on_approval`.
- **«К оплате»** — заявки с `executionStatus = awaiting`, отдельной строкой
  «отдельно просрочено N» (`overdue`).
- **«Платёжки»** — ждут подтверждения (`awaiting_confirmation`), ошибок
  (`failed`), отправлено сегодня (`sent`, `sentAt` в текущих московских сутках).

Ссылки карточек: «На счетах» → `/reference/bank-accounts`, «На согласовании»
и «К оплате» → `/requests`, «Платёжки» → `/dispatch`.

### 2. График движения денег (полная ширина)

- Столбчатый график shadcn charts: две серии по дням — поступления
  (сумма положительных `amountMinor`) и списания (модуль суммы отрицательных).
  Столбцы рядом (grouped, не стопкой). Цвета — токены темы `--chart-1` / `--chart-2`.
- Переключатель периода 7 / 30 / 90 дней — `toggle-group` над графиком.
  Сервер отдаёт сразу 90 дней, переключатель фильтрует на клиенте без
  запросов к серверу (по образцу официального интерактивного примера shadcn).
- Клиентский компонент графика лежит рядом со страницей (`app/`), данные
  получает пропсами. Подписи сумм — `formatMoney`, даты — `formatDate` (Москва).

### 3. Таблица «Остатки по счетам» (Card + Table)

Колонки: организация, банк, счёт, валюта, остаток. Валютные счета — остаток
в валюте плюс пересчёт в ₽ по `CurrencyRate`. Сортировка: организация, затем
название счёта.

### Данные и доменная логика

- Счётчики заявок и платёжек — Prisma `groupBy` по статусу.
- Транзакции — выборка за 90 дней (только `occurredAt`, `amountMinor`),
  группировка по дням — в домене.
- Новая доменная логика в `lib/domain/` (чистые функции, без Prisma и React,
  обязательные unit-тесты рядом):
  - в `lib/domain/transactions.ts` — группировка транзакций по московским
    суткам с суммами прихода/расхода за N дней;
  - новый модуль `lib/domain/balances.ts` — итог по счетам в ₽ с пересчётом
    валют по курсам. Нет курса для валюты счёта → счёт показывается с пометкой
    «нет курса», в итог не входит, итог помечается как неполный.

### Деньги

- Остатки (`AccountBalance.balanceMinor`) — `BigInt`: суммирование и
  форматирование (`formatMoneyBig`) на сервере, в клиентские компоненты
  уходят готовые строки (BigInt в клиент передавать нельзя).
- Дневные суммы графика — целые копейки числом (агрегаты `Int`-транзакций);
  форматирование — только `formatMoney` / `formatMoneyBig`, float для денег
  не используется.

## Пустые состояния и ошибки

- `AccountBalance` пуст (срез 1С не синкался): карточка «На счетах» — серое
  «нет данных» (тот же принцип, что у проверок светофора), вместо таблицы
  остатков — заглушка «Данные ещё не синхронизированы».
- Нет транзакций за выбранный период — на месте графика «Нет операций за период».
- Нулевые счётчики заявок/платёжек — просто нули, это штатно.
- Мутаций и server actions на дашборде нет, паттерн FormState не задействован.
  Неожиданные сбои ловит существующий глобальный `app/error.tsx` — ничего
  нового не строим.

## Тесты

- **Unit (Vitest, TDD — тест до кода)** для обеих доменных функций:
  - группировка по дням: границы московских суток, разделение прихода/расхода,
    пустой список, транзакции вне периода;
  - итог по счетам: рублёвые и валютные счета, отсутствующий курс (счёт вне
    итога, пометка неполного итога), арифметика BigInt.
- **E2e-смоук (Playwright, `tests/e2e/`)**: логин → сайдбар с группами виден →
  на дашборде есть stat-карточки, график и таблица → клик по «Транзакции»
  ведёт в раздел, активный пункт подсвечен. Отдельная проверка: у читателя
  нет группы «Настройки». Данные создаются внутри теста, без опоры на seed.
- **Ручная проверка в браузере** до коммита: обе темы, мобильная ширина,
  чистая консоль (ошибка `nativeButton` у Base UI видна только в живой консоли).

## Вне рамок

- Переключатель периода через URL, drill-down по графику, виджеты фондов
  (`FundSnapshot`) и статистики партнёров — не в этой итерации.
- Право на просмотр разделов не меняется: скрытие пунктов сайдбара — UI-фильтр
  поверх существующих проверок страниц.
